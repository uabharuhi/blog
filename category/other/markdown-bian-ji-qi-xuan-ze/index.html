markdown 編輯器選擇

  • harapod
  • sublime Text 3 + 有的沒的插件

因為要架設static blog使用markdown寫文章, 不太想用web online的editor(個人偏好), 所以想說找幾款可以在windows上面使用的markdown editor, 因為需要記錄一些讀書的心得會使用到mathjax之類的東西,最好也要能快速的preview,界面也不要太醜比較好。找來找去比較喜歡的只有haropad。

haropad

界面很漂亮,還可以選擇各式各樣的Theme,又可以用mathjax,但是在我的電腦上(win7 desktop)上用起來莫名的卡...... 而且第一次用的時候調完設定之後mathjax inline mode還是有問題,結果重新打開haropad之後又正常了,但是速度的問題一直很漫,最後放棄治療...到底是為什麼阿會那麼卡

因為真的太卡,想說找一個輕量的editor,但是可以裝plugin滿足各式各樣的需求,網路上的人各種推sublime,我之前已經聽過好幾遍了,想說就用用看好了。

去官網下載安裝完之後sublime,開始裝plugin,達到我的需求需要裝以下幾個plugins 在那之前還要先裝 Package Control

  1. Package Control
  2. Markdownediting
  3. Markdown-preview
  4. LiveReload-sublimetext3

Package Control

安裝非常簡單,把官網上一段程式碼貼到sublime console(ctrl+` )然後按下enter就弄好了

Markdownediting

這個插件是讓markdown的寫作更方便,例如會新增好用的hotkey,字動補全等等

利用package control的install package參照github上面的要求簡單搞定,真的有夠方便

Markdown-preview

裝完Markdownediting後,接下來要裝這個,是為了方便即時顯示寫出來的markdwon,我自己的期望是能夠在editor內部就能夠顯示...問題是好像找不到,用瀏覽器也沒多差拉算了(搞不好因為這樣所以才不會卡)

安裝方式也是按照github上面隨便弄一弄就裝起來了,有夠爽 為了方便顯示的操作,設定hotkey,github上面也有寫

or bind some key in your user key binding, using a line like this one: { "keys": ["alt+m"], "command": "markdown_preview", "args": {"target": "browser", "parser":"markdown"} }, for a specific parser and target or { "keys": ["alt+m"], "command": "markdown_preview_select", "args": {"target": "browser"} }, to bring up the quick panel to select enabled parsers for a given target.

這樣就能夠按下ctrl+M開啟一個網頁顯示markdown的結果,不用搞那些有的沒的...

另外,還要啟用mathjax之類的功能,要去Preferences->pakage setting->markdwon preview調設定檔,再user setting加入

{
"enable_mathjax": true,
"enable_highlight": true,
"enable_autoreload": true
}

這樣應該就沒問題了

LiveReload

沒有安裝這個的話,每一次markdown改完之後就要用F5或是重開一個刷新頁面,很煩.... 裝下去之後,儲存的同時會自動刷新頁面,要用這個功能chorme也要裝一個autoreload的插件,裝下去之後就爽爽用了

因為安裝的時候弄得有點複雜,不知道先裝什麼後裝什麼,其實只要按照這個安裝plugins的順序,就很簡單了,可能有什麼遺漏的地方就要發揮靈性了.... 應該有把該紀錄的重點都記錄起來了
